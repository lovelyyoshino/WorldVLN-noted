# WorldVLN Post-Training

This directory contains the WorldVLN post-training workflows used for StageA rollout collection, replay metadata generation, local inference integration, and StageB partial-freeze optimization.

## Overview

The post-training pipeline is organized around two stages.

- StageA collects candidate rollouts through a remote simulator-backed online RL workflow.
- StageB consumes the replay metadata produced by StageA and continues optimization from a pretrained checkpoint.

This package also includes the local inference service used by StageA and the simulator-side runtime needed for remote simulator rollout.

## Key Components

| Path | Purpose |
| --- | --- |
| `InfinityStar-main/` | Core training code used by StageA and StageB. |
| `TSformer-VO-main/` | Action-head dependency used by the local inference service. |
| `infinity_tsformer_api_server.py` | Local inference module used during rollout collection. |
| `runtime/infinity_tsformer_client.py` | Simulator-side client and service implementation for `remote_sim`. |
| `scripts/run_stagea_collect.sh` | Public StageA launcher. |
| `scripts/run_stageb_partialfreeze.sh` | Public StageB launcher. |
| `scripts/run_remote_sim_service.sh` | Public simulator service launcher. |
| `run_infer_server.sh` | Public local inference server launcher. |
| `outputs/` | Default output root for rollout caches, replay metadata, logs, token caches, and checkpoints. |

## Setup

This package does not ship private checkpoints or dataset payloads. You must provide the required assets through environment variables or explicit command arguments.

| Asset | Variable | Purpose |
| --- | --- | --- |
| Rollout source manifest | `SRC_JSON` | StageA rollout source definition. |
| InfinityStar checkpoint | `INFINITY_CKPT` | Local inference for StageA and related rollout flows. |
| Shared model assets | `CHECKPOINTS_DIR` | T5 and VAE assets used by StageA, local inference, and StageB. |
| Action-head checkpoint | `ACTIONHEAD_CKPT` | Action prediction head for rollout inference. |
| Action-head config | `ACTIONHEAD_RUN_CONFIG` | Matching runtime config for the action head. |
| Resume checkpoint | `RUSH_RESUME` | Optional initialization checkpoint for StageB. |
| Task JSON root | `UAVFLOW_TASK_JSON_ROOT` | Task definitions required by `remote_sim` StageA. |
| Replay metadata | `REPLAY_META_DIR` | StageB input generated by StageA. |

## StageA: Rollout Collection

StageA builds rollouts and replay metadata for the online RL pipeline from a rollout manifest plus model assets.

### Inputs

- A rollout source manifest provided through `SRC_JSON`
- Model checkpoints and shared assets
- Simulator connection settings for `remote_sim`

### Outputs

- Rollout caches under `outputs/rlcache/`
- Replay metadata under `outputs/offline_grpo_data_fast/`
- Local logs under the configured output root

### Start the Local Inference Service

```bash
INFINITY_CKPT=/path/to/infinity/global_step_xxx.pth \
CHECKPOINTS_DIR=/path/to/checkpointsinf \
ACTIONHEAD_CKPT=/path/to/actionhead/checkpoint_last.pth \
ACTIONHEAD_RUN_CONFIG=/path/to/actionhead/run_config.json \
bash run_infer_server.sh
```

### Run StageA

```bash
SRC_JSON=/path/to/reference_video_full_49f_trajectory_prompts.json \
INFINITY_CKPT=/path/to/infinity/global_step_xxx.pth \
CHECKPOINTS_DIR=/path/to/checkpointsinf \
ACTIONHEAD_CKPT=/path/to/actionhead/checkpoint_last.pth \
ACTIONHEAD_RUN_CONFIG=/path/to/actionhead/run_config.json \
UAVFLOW_STAGEA_ROLLOUT_BACKEND=remote_sim \
UAVFLOW_SIMULATOR_BASE_URL=http://127.0.0.1:18765 \
UAVFLOW_TASK_JSON_ROOT=/path/to/UAV-Flow-Eval/test_jsons \
bash scripts/run_stagea_collect.sh RUN_ID=remote_sim_smoke TOP_N=1 K_CAND=1 STAGEA_NPROC=1
```

## StageB: Partial-Freeze Optimization

StageB consumes replay metadata produced by StageA and performs post-training updates.

### Inputs

- `REPLAY_META_DIR` generated by StageA
- `CHECKPOINTS_DIR` for shared model assets
- `RUSH_RESUME` when resuming from a pretrained checkpoint

### Outputs

- Training logs under the configured train log root
- Updated checkpoints under the configured checkpoint root
- Token caches under the configured token cache root

### Run StageB

```bash
CHECKPOINTS_DIR=/path/to/checkpointsinf \
RUSH_RESUME=/path/to/infinity/global_step_xxx.pth \
REPLAY_META_DIR=/path/to/replay_meta_stagea_smoke \
bash scripts/run_stageb_partialfreeze.sh PARTIAL_FREEZE_MODE=smoke RUN_ID=stageb_smoke
```

## Remote Simulator

The `remote_sim` path keeps policy inference on the training host and delegates environment reset, action execution, and rendering to a simulator host.

Start the simulator-side service with:

```bash
python runtime/infinity_tsformer_client.py \
  --mode service \
  --host 0.0.0.0 \
  --port 8765 \
  --task_json_root /path/to/UAV-Flow-Eval/test_jsons
```

If the training host cannot directly reach the simulator host, use reverse SSH port forwarding and point `UAVFLOW_SIMULATOR_BASE_URL` to the forwarded localhost port. See `docs/remote_sim.md` for the request flow and runtime expectations.

## Notes

- Default outputs are written under `outputs/` unless overridden.
- `config.json` is publish-safe by default; checkpoint fields are intentionally left empty and should be supplied through runtime configuration.
- The packaged StageB trainer includes the small-smoke profiling guard so tiny replay metadata runs do not divide by zero.
